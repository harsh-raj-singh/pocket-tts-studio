# Contributing to Pocket TTS Studio

Thank you for your interest in contributing! Here are some guidelines to help you get started.

## Development Setup

```bash
git clone https://github.com/harsh-raj-singh/pocket-tts-studio.git
cd pocket-tts-studio
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

## Code Style

- Follow PEP 8 conventions
- Use type hints for function signatures
- Keep functions focused and single-purpose
- Use descriptive variable names

## Pull Request Process

1. Create a feature branch from `main`
2. Make your changes with clear, descriptive commit messages
3. Test your changes manually (both web UI and CLI)
4. Update documentation if you've changed behavior
5. Open a PR with a clear description of the change and motivation

## Bug Reports

When filing an issue, please include:

- Python version and OS
- Steps to reproduce
- Expected vs actual behavior
- Any relevant logs or error messages

## Feature Requests

We welcome feature suggestions! Please describe the use case and expected behavior clearly.
